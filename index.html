THE OPEN SOURCE UNIVERSE

Never contributed before? This 3D simulation shows exactly how open source works. Hover over objects to learn.

Central Repository
Forks (your copy)
Merged PRs
Open Issues
📚

Hover over any object in the simulation to learn what it represents in real open source projects.

1

🟣 The Repository (Repo)

Think of it as a project folder on GitHub. It contains all the code, history, and issues. The big purple sphere in the center IS the repo.

2

🔵 Fork = Your Personal Copy

When you fork a repo, GitHub creates YOUR OWN copy. The cyan orbiting spheres are forks — they spin around the main repo.

3

🟢 Pull Request (PR)

You make changes in your fork, then open a "Pull Request" asking the main repo to accept your changes. Green streaks = PRs flying toward the center.

4

🟠 Issues

Issues are tasks, bugs, or feature requests. Orange orbs are open issues waiting for someone (YOU!) to claim and fix them.
